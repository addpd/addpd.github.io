一、安装hexo

安装 Hexo 和了解其目录结构是使用 Hexo 搭建博客的第一步。下面是安装 Hexo 的步骤、目录结构的介绍以及如何添加文章的详细说明。

1. 安装 Hexo

前提条件：

2. Hexo 目录结构

Hexo 项目目录结构如下：

重要目录和文件说明：

  • _config.yml：主配置文件，所有网站的配置都在这里设置，如博客标题、作者、语言等。
  • scaffolds：文章模板，可以在这里定义文章、页面和草稿的默认格式。
  • source：存放你的文章、页面和静态资源（如图片、CSS、JavaScript 等）。
    • _posts：这是存放博客文章的地方，每篇文章都是一个 Markdown 文件，Hexo 会自动将其转换为 HTML 页面。
    • _drafts：这是存放草稿的地方，草稿不会被发布，直到你将它移动到 _posts 目录下或使用命令发布。
  • themes：主题目录，Hexo 支持主题更换，你可以在 themes 目录下添加不同的主题。

在文章的 Front Matter 中，你可以为每篇文章设置 title、description 和 keywords，这有助于 SEO。你还可以通过 Hexo 插件来自动生成 sitemap.xml 和 robots.txt，让搜索引擎更好地抓取你的网站。

总结

  1. 安装 Hexo：通过 npm 安装并初始化 Hexo 博客。

这个错误信息表明 Git 检测到当前用户与仓库目录的拥有者不一致，可能存在安全风险。为了让 Hexo 能够正确部署，你需要告诉 Git 这个目录是安全的。具体操作如下：

为什么需要这一步

  • Git 安全目录检查：Git 2.35 之后增加了一个安全检查，防止用户在不受信任的目录中执行命令。这是为了防止攻击者通过操纵 Git 仓库目录来执行恶意代码。
  • 权限不一致：错误中显示 .deploy_git 目录的所有者是 BUILTIN/Administrators，而当前操作的用户是 addpd。通过添加安全目录，可以告知 Git 即使权限不一致也可以继续操作。
