控制台出现报错：The Content Security Policy directive 'frame-ancestors' is ignored when delivered via a <meta> element.

解决，meta中去掉frame-ancestors

你看到的错误提示 **”The Content Security Policy directive ‘frame-ancestors’ is ignored when delivered via a

element”**，是因为 frame-ancestors 指令不能通过 <meta> 标签来设置。

原因：

  • frame-ancestors 是用来控制哪些网站可以通过 <iframe> 嵌入当前页面的，它必须通过 HTTP 响应头设置，而不能通过 <meta> 标签设置。浏览器出于安全原因忽略了在 <meta> 标签中使用 frame-ancestors，只能通过服务器级别的响应头来强制执行这个策略。

解决方案：

要正确使用 frame-ancestors，你需要在服务器配置中添加它，而不是通过 <meta> 标签。

解释：

  • **frame-ancestors 'self'**：只允许当前域将页面嵌入到 <iframe> 中。
  • **https://example.com**：允许特定的外部域嵌入。

小结：

如果你希望使用 frame-ancestors，需要将其添加到服务器的响应头中，而不是通过 HTML 中的 <meta> 标签。这是因为浏览器只会在通过服务器响应头设置时执行此指令。
